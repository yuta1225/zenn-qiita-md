# 実務に活かせるか

AWS SAA 勉強したけど、ぶっちゃけ実務に活かせるかというと微妙だと思う。
クライアントの要件に合わせて技術選定するところとかはどれだけインプットこなしても実務には勝らない。
ただし、細かいところで実装のスピードが上がるのは実感できた。

IAMポリシーとかロールのアタッチとかは概念理解するとすんなりEKS周りの実装が早くなった。
あと、構築のときに必要なパラメータとかはパっと思いつくので、整理しやすい。

やって損することはない。

だから、実務に活かしたいなら実務をガチった方が定着は早い。
どちらかというと、マインドとしては自信を付けたり知識を整理するために勉強した方がいい。

# 学習するときのポイント

意外とストリーミング系（Kinesis）とかの話とか出てくる。
AWSについての全般的なサービスについては知っておいた方がいい。
ただ、その内容は割とCLFに準拠しているので、CLFを受験しておくとすんなりいけるかも。

実装するときの細かい仕様とかも問われる。

case1: dynamodbの細かい仕様
DynamoDBの一貫性モデルの種類
Eventually Consistent Reads（最終的一貫性）

標準設定
多少遅れても、すべての読み取りが最新の状態に追いつく
パフォーマンスが高い
Strongly Consistent Reads（一貫性のある読み取り）

必ず最新のデータを返す
少し遅延が増加する可能性がある
重要なトランザクションや整合性を重視する場合に使用

case2:EFSのファイルシステムって何的な
Amazon Elastic File System（EFS）が使用している通信プロトコルはどれか。

SMB（Server Message Block）
NFS（Network File System）
AFP（Apple Filing Protocol）
iSCSI

上記はまだ浅いが、もっと深いところまで問われた気がする。

しかし、基本的にはオートスケーリングだとか権限とか、割とベスプラ的なところを問われるので、そこさえできていれば5割ぐらいはいけると思う。
EC2, VPC, autoscaling, sqs, S3, IAM, Lamda周り
